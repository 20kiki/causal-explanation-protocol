# 通用因果解释协议

**Language:** [English](../README.md) | [简体中文](README.md)

一个结构化的严谨因果解释协议。先分类，再解释。先审计驱动力，再构建链条。

## 它解决什么问题

在解释"为什么X会发生"时，很容易陷入循环论证、错误类比、或伪根本因。本协议提供：

- **前置检查** —— 陷阱识别 + 驱动力溯源审计，在任何解释开始前强制执行
- **四模式分流决策树** —— 在构建解释前，将每个现象分类到四种因果结构之一
- **每模式专属模板** —— 每种模式有特定的解释形态、起始因定义、和类比要求
- **经过实证测试** —— 按 TDD 方法构建：先让未加载 skill 的 agent 执行因果解释任务，记录系统性失败，再针对性地编写协议修复所有失败点

## 四种解释模式

| 模式 | 适用对象 | 核心操作 |
|------|---------|---------|
| **A: 必然传导链** | 被动、强约束的物理/工程系统 | 从独立守恒律出发，构建单向无分支链条 |
| **B: 稳态博弈链** | 负反馈系统、规则锁定的博弈 | 将循环打包为稳态模块（禁止逐段解包辩护） |
| **C: 路径依赖链** | 历史锁定、初始条件敏感 | 分离偶然性（不解释"为何选中B"）与必然性（解释"为何锁定不可逆"） |
| **D: 涌现自组织链** | 大量个体遵循简单规则 | 强制执行微观→宏观的因果单向性 |
