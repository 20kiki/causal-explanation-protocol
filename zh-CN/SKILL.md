# 通用因果解释协议

## 概述

不是教你怎么做因果推理——而是提供一个**前置分流 + 结构化模板**，确保每次解释前执行相同的预检查，根据现象类型选择正确的解释形态，避免循环论证和错误类比。

核心原则：**先分类，再解释。先审计驱动力，再构建链条。**

## 使用时机

- 用户问"为什么X会发生？"或"X是怎么产生的/形成的/出现的？"（与"为什么X"等价的因果问题）
- 需要向他人解释某个现象的因果机制
- 审查或反驳一个已有的因果解释
- 系统涉及反馈循环、路径依赖、涌现、或均衡状态 —— 但也适用于简单物理传导链（模式 A 就是为这个而存在的）
- 怀疑解释中存在循环论证或伪类比

## 前置检查层（强制执行）

在构建任何解释之前，必须执行以下两步。**不可跳过。**

### 第一步：陷阱识别

扫描解释空间中是否存在以下谬误：

| 谬误类型 | 检测问题 | 危险信号 |
| :------- | :------- | :------- |
| 循环论证 | "因"是否需要"果"来定义自身？ | 两个变量互为定义 |
| 错误类比 | 类比的因果结构与目标系统是否同构？ | 表面相似但机制不同 |
| 伪根本因 | 声称的"因"本身是否还有更深层的驱动力？ | 答案可以无限追问"为什么" |

### 第二步：驱动力溯源审计

对任何声称的"因"，追问：

```
这个"因"的驱动力是什么？
  ├── 主动意图（设计、决策、目的性行为）
  ├── 被动约束（物理定律、守恒律、外部边界条件）
  └── 涌现规律（大量个体的统计必然性）
```

只有追到上述三类之一，才能宣称找到了"根本因"。如果还能继续追问"为什么"，说明你还没到底。

## 分流决策

完成前置检查后，根据以下决策树将现象归入四种模式之一：

```
系统是否包含负反馈环（稳态维持）？
  ├── 是 → 模式 B：约束下的稳态博弈链
  └── 否 → 系统的结果是否受初始条件强烈影响且存在正反馈锁定？
            ├── 是 → 模式 C：路径依赖的演化/历史链
            └── 否 → 系统的宏观模式是否由大量个体遵循简单规则产生？
                      ├── 是 → 模式 D：涌现与自组织链
                      └── 否 → 模式 A：必然传导链
```

## 四种解释模式

### 模式 A：必然传导链

**适用**：稳态、被动、强约束的非意图系统（物理、工程）。

**起始因性质**：绝对外在、不变的物理边界或守恒律。

**解释形态**：
```
因A（独立物理边界/守恒律）→ 被动导致B → 必然导致C
```

强要求：
- 链条单向、无分支、无循环
- 每一步必须从前一步必然导出
- 不允许任何形式的循环引用

**类比要求**：必须用同构物理机制的系统类比（如划船桨拨水 → 机翼推空气），而非表面相似。

---

### 模式 B：约束下的稳态博弈链

**适用**：含负反馈的稳态系统、有意图但被规则锁定的博弈。

**起始因性质**：一组相互制衡的规则或力，共同构成一个稳定吸引子。其中至少有一个元素是外生的（不由系统内部状态决定）。

**核心操作——打包而非解包**：

❌ **错误做法**（解包循环）：
> "温度低→加热→温度升高→停止加热→温度降低→又回到温度低……每一步都是单向的所以不循环。"

这虽然逻辑上可辩护，但浪费了解释力。用户想知道的不是"每一步为什么合理"，而是"整个系统为什么必然稳定在某个值附近"。

✅ **正确做法**（打包为稳态模块）：
> "系统由X和Y两种相互制约的力/规则共同定义。X推动系统向一个方向，Y的强度随偏离程度而增大。两者相交的点就是稳态Z。一旦偏离Z，Y的约束力自动增强将其拉回。因此Z是必然的稳态结果。"

**解释形态**：
```
规则集合{X, Y}定义稳态Z → 数学/物理上Z是唯一交点 → 系统必然在Z附近波动
```

**类比要求**：必须用同样具有"冲突中的均衡"结构的系统类比（如弹簧吊重物：重力与弹力抗衡，系统自动稳定在某个长度），不能用单向传导链的系统类比。

---

### 模式 C：路径依赖的演化/历史链

**适用**：形态/制度是历史累积结果，且初始条件敏感。

**起始因性质**：分叉点上的微小差异 + 正反馈放大机制。两部分缺一不可。

**关键边界划分**：

| 要解释的（必然性） | 不解释的（偶然性） |
| :----------------- | :----------------- |
| 一旦上了某条轨道，为什么很难回头并必然走向C | 为什么偏偏选中了B而不是A |

**解释形态**：
```
分叉点：微小差异/随机波动 → B被选中（这属于偶然，不解释为什么是B）
锁定机制：B → 触发正反馈M（规模效应/网络效应/习得性投资）→ 可能性空间急剧收窄 → 必然导向C
```

**类比要求**：必须用同样具有"分叉 + 锁定"结构的系统类比（如滚雪球下山：微小初始颠簸决定最终路径），不能用只有锁定没有分叉的系统类比。

---

### 模式 D：涌现与自组织链

**适用**：大量简单个体遵循简单规则，产生复杂宏观模式。

**起始因性质**：底层的个体规则（算法）。这些规则是固化、外在的。

**核心约束——因果箭头不可逆**：
```
微观规则R → 无数并行互动 → 宏观统计规律P
```
P不能反过来解释R。宏观模式是描述性的，不是因果性的。

**解释形态**：
```
所有个体独立遵循规则R → 集体行为在数学上必然产生宏观模式P → 无需中央协调
```

**检测是否违规**：如果解释中出现"因为拥堵波存在，所以司机减速"这种宏观→微观的因果箭头，就是循环论证。正确说法是"每个司机因前车减速而减速，这些局部决策的集合形成了拥堵波"。

**类比要求**：必须用同样基于并行规则涌现的系统类比（如沙堆雪崩、磁针集体指向），不能用有领导者或中央协调的系统类比。

## 常见错误（来自实际测试）

| 错误 | 正确做法 |
| :--- | :------- |
| 跳过前置检查直接解释 | 必须先执行陷阱识别 + 驱动力溯源，不可跳过 |
| 不分类就开始解释 | 必须先用分流决策树确定模式，再按对应模板构建解释 |
| Mode B中解包循环而非打包 | 停止逐段辩护。找到外生锚点 + 制衡关系，打包为稳态模块 |
| Mode C中试图解释"为什么是B不是A" | 明确声明这属于偶然性，解释力应用于"为什么锁定"而非"为什么选中" |
| Mode D中用宏观模式解释微观行为 | 检查每个因果箭头的方向，确保微观→宏观，而非反之 |
| 类比未经同构检查 | 对比目标系统和类比系统的因果结构（反馈类型、驱动力来源），而非表面特征 |

## 快速参考

| 现象特征 | 对应模式 | 起点形式 | 核心操作 |
| :------- | :------- | :------- | :------- |
| 无反馈、被动物理过程 | A: 必然传导链 | 独立物理边界/守恒律 | 构建单向无分支链条 |
| 有负反馈、稳态维持 | B: 稳态博弈链 | 相互制衡的规则集合（含外生锚点） | 打包循环为稳态模块 |
| 有正反馈、历史锁定 | C: 路径依赖链 | 分叉点微小差异 + 放大机制 | 分离偶然性（不解释）与必然性（解释） |
| 大量个体、简单规则 | D: 涌现自组织链 | 底层个体规则 | 保证微观→宏观的因果单向性 |

## 特定领域增强（可选声明）

### 对计算机/网络/分布式系统：分层因果锁定

计算机系统的独特风险：同一个可观测症状（如"服务挂了"）的因果结构完全取决于你锁定的抽象层级。用错误的模式解释一个层级，是**范畴错误**，不是解释质量问题。

在进入分流决策前，必须先声明分析层级：

```
“本解释锁定在 [物理层 / 内核层 / 协议层 / 应用层 / 业务层]，
仅讨论该层内部的因果机制，不跨层混淆驱动力的类型。”
```

**为什么必须这样做**：

| 同一症状"服务不可用" | 锁定层级 | 对应模式 |
| :------------------- | :------- | :------- |
| 光纤被挖断 | 物理层 | A：必然传导链（物理信号中断→链路断开→路由不可达） |
| 网络分区，CAP 定理下 CP 系统拒绝写入 | 协议层 | B：稳态博弈链（一致性 vs 可用性相互制衡，Paxos/Raft 在多数派存活下维持稳态） |
| 死锁（两个事务互相等待对方持有的锁） | 内核/应用层 | B：稳态博弈链（锁的循环等待作为"吸引子"，无外部干预永不解除） |
| 重试风暴（超时→重试→更多超时→更多重试） | 应用层 | D：涌现自组织链（每个客户端独立遵循重试规则，涌现雪崩） |

不声明层级就进入分流 → 可能用 Mode D 解释物理断网，或 Mode A 解释 CAP 脑裂。两者都错。

### 对极复杂现象：模式叠加（严格守卫）

极少数现象同时跨越两种因果结构的边界。此时可允许模式叠加，但**必须满足以下全部条件，缺一不可**：

```
1. 每个模式只解释现象的 [一个独立侧面]，不可重叠。
   例：Mode C 解释"锁定如何发生"，Mode B 解释"锁定后的均衡为何自我维持"。

2. 必须显式声明每个模式覆盖的因果范围和时间边界。
   例："[Mode C 覆盖 1870–1930：路径如何被选中并锁定]
         [Mode B 覆盖 1930–现在：锁定后的 Nash 均衡如何自我维持]"

3. 叠加模式下，每种模式的"起始因"必须独立定义，不可共用。
   违反例："起始因同时是历史偶然性和市场均衡力量" ← 不允许。

4. 如果无法清晰分离两个侧面 → 回到分流决策树，选主导模式。
   不确定时，默认只用一种模式。
```

**未满足以上任一条件时，禁止使用叠加。不确定时，禁止使用叠加。**
